Test code is the spec — write it as given (adapting to the exact harness helpers).

**Goal:** Add `ForMatch` — the consumer-facing mirror of `ForAttributeWithMetadataName` — so a Roslyn generator author can hook their incremental pipeline onto a Synto `[Match]` pattern.

**Architecture:** Per pattern, Synto emits a cheap companion predicate `XCouldMatch(SyntaxNode)` and a descriptor `MatchPattern<XMatch> XPattern`. A small self-contained surface (`Matched<T>`, `MatchPattern<T>`, `SyntoMatchProviderExtensions.ForMatch`) is injected into the consumer's generator project via the existing surface-injection path. `ForMatch` wraps `SyntaxValueProvider.CreateSyntaxProvider(predicate: CouldMatch, transform: Match)`; a thin overload yields the match (convenient, not cacheable) and a projecting overload runs matcher + consumer projection in one transform (cacheable by construction).

**Tech Stack:** C# / .NET, Roslyn incremental source generators (`IIncrementalGenerator`), xUnit v3 + Verify snapshots, jujutsu (jj) VCS.

**Spec:** `docs/superpowers/specs/2026-06-21-formatch-design.md` (contracts C-FM1…C-FM4).

## Global Constraints

Every task implicitly includes these (verbatim from the spec / v1 contracts):

- **Self-contained, no runtime dependency.** Injected helpers + generated members compile on **`netstandard2.0`** with no Synto runtime-package reference. Records/`init` members rely on the `IsExternalInit` polyfill the matching DSL already injects. (C-FM3 / v1 C3.)
- **Equatable pipeline.** No `SyntaxNode` / `SemanticModel` / `ISymbol` may enter **cached** pipeline state. The projecting `ForMatch` overload's only pipeline output is the consumer's `TResult`. (C-FM2 / v1 C4.)
- **Cacheability anti-pattern (do NOT do this).** The projecting overload must **not** be implemented as `thinOverload.Select(transform)` — that caches the non-equatable `Matched<TMatch>` at the intermediate step. Matcher + consumer projection run in **one** `CreateSyntaxProvider` transform.
- **Predicate superset.** `XCouldMatch(n)` must return `true` for every node where `X(n) != null`. (C-FM1.)
- **Green gate (run before every commit):** `dotnet build --no-restore -c Debug` (0 errors; analyzer warnings are findings, not gate failures) · `dotnet test --no-build -c Debug` (all green) · `dotnet format whitespace` then `dotnet format whitespace --verify-no-changes` (no diffs). **Whitespace scope only** — never full `dotnet format` (it applies CA1515/CA1815 rewrites that break the build).
- **Commits:** Conventional Commits, **no AI/Claude footer**, one commit per task. VCS is jj: `jj commit -m "…"` finalizes `@` and opens a new change on top.
- **Branch:** work on `experimental/matching` (or a sibling `experimental/*`). Never main; no `jj git push` without explicit instruction. Do not move the `experimental/matching` bookmark; the operator advances it after review.
- **Locked names:** `ForMatch`, `Matched<TMatch>`, `MatchPattern<TMatch>`, `SyntoMatchProviderExtensions`; generated members `{Pattern}CouldMatch`, `{Pattern}Pattern`. Namespace `Synto.Matching`.
